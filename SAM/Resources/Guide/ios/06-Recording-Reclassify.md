# Reclassifying a Recording

SAM uses different summary prompts for Client Meetings, Training/Lectures, and Board Meetings — they extract different fields (action items vs. learning objectives vs. votes). If you picked the wrong type up front, you can change it once the recording finishes.

## How to Use

1. After tapping Stop, wait for the summary to land in the completed view.
2. Below the summary, tap **Change Recording Type** (the pill with an icon and ⌃⌄ chevrons).
3. Pick the correct type from the menu. A confirmation dialog asks you to confirm — tap **Resummarize**.
4. SAM discards the existing summary and the Mac regenerates it using the new prompt. This takes 10–30 seconds.

## Available Types

- **Client Meeting** — action items, follow-ups, decisions, life events, open questions
- **Training/Lecture** — key points, learning objectives, review notes
- **Board Meeting** — attendees, agenda, votes, decisions

## When the Option Disappears

The **Change Recording Type** button is only available while the recording is still owned by both devices. It hides as soon as:

- You tap **Done** (sign-off purges the verbatim transcript on the Mac).
- The Mac is no longer reachable on the network.
- Reclassification is already in progress.

After Done is tapped, the only place to relabel is on the Mac.

## Tips

- Reclassification can't be undone. The previous summary is discarded the moment you confirm.
- If the live recording was a board meeting but you accidentally started it as Client Meeting, reclassify before signing off — otherwise the votes and agenda items won't be extracted.
- The Mac re-runs only the summary step; transcription and diarization aren't re-done.
